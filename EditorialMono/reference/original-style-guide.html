Editorial Mono

A Swiss editorial system for thingsithinkithink. Three families, one accent, sharp corners. Mono carries structure — labels, captions, indices. Serif carries prose. Sans carries UI chrome. Vermilion is the only voice raised above the page.

01  ·  Principlesfoundations

Why this system exists

Personal blogs default to one of two failure modes: too generic to remember, or too ornamented to read. V2 picks a third path — read like a working notebook. Hierarchy through space and weight, not boxes and color. The accent is a voice, not decoration.

Structure first.
Mono labels everything before prose begins.
One accent.
Vermilion is the only color that raises its voice.
No rounding.
Every corner is square. Every rule is hairline.

07  ·  Patternspage templates

Three pages, one system

The system supports the index, the post, and the archive/tags page. Each is the same rail + same content padding + same vermilion logic — only the content slot changes.

INDEX
Rail nav highlights "Index" · content area = list of post cards · meta strip across top reads "ARCHIVE · 142 POSTS · UPDATED FEB 15"
POST
Rail nav collapses to current post (title + TOC + tags + prev/next) · content = h1 + dek + author + hero + prose
TAGS / ARCHIVE
Rail nav highlights tag · content = h2 per year + monospace year-month-title row, hover shows accent
08  ·  Do / Don'tedges of the system

The edges

DO
Use vermilion for the §, the drop cap, and the link underline. Three places, no more.
DON'T
Don't use vermilion for buttons, banners, callouts, or "important" backgrounds. It loses meaning the moment it's everywhere.
DO
Keep all corners square. Hairline rules only. Empty space is the system's primary tool.
DON'T
No rounded corners. No drop shadows. No card-with-3px-border-and-glow. Those belong to a different system.
DO
Stamp every figure with FIG · in the mono caption. Stamp every meta strip with the eyebrow.
DON'T
Don't add a fourth font. Don't add icons. The mono is doing all the iconographic work the page needs.
09  ·  Using the systemhandoff

Building a new post

  1. Load tokens.css then components.css.
  2. Wrap the page in <body class="v2"> · <div class="v2-page">.
  3. Left column is <aside class="v2-rail"> — populate brand, nav, post metadata, TOC, tags, prev/next.
  4. Right column is <article class="v2-article"> — meta strip, h1, dek, author, hero, prose.
  5. Inside .v2-prose, use .v2-h2, .v2-h3, .v2-quote, .v2-code, .v2-figure as needed.
  6. Close with .v2-end.

If you need a new component, ask: does it fit the three principles (structure first, one accent, no rounding)? If yes, propose it. If no, you probably want a different system.
